# Feature 1.1.8: All Pydantic Schemas Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Define every Pydantic request/response schema in `backend/schemas/` so the repository, service, and router layers can type their data in Phase 2-6 without rewriting them.

**Architecture:** One file per domain (seat, session, invoice, member, etc.). Each file exports a `*Base` (shared fields), `*Create` (required for insertion), `*Update` (all-optional partial update), and `*Response` (full entity, `model_config = ConfigDict(from_attributes=True)`). Pure Pydantic -- no DB access.

**Tech Stack:** Pydantic v2 (`pydantic.BaseModel`, `pydantic.ConfigDict`, `pydantic.Field`, `pydantic.AwareDatetime`), Python 3.12+, `backend.models._enums` for enum reuse.

## Global Constraints

- All monetary fields: `int` (paise) -- **never `float`** (R-06). Pydantic `int` type rejects `float` automatically.
- All string fields: `pydantic.Field(..., max_length=N)` to prevent oversized inputs.
- All timestamps: `pydantic.AwareDatetime` (enforces timezone-aware `datetime` objects).
- Response schemas are separate from request schemas (no `pin_hash` leakage).
- Every response schema sets `model_config = ConfigDict(from_attributes=True)` for clean ORM -> Pydantic conversion.
- Enum fields reuse `backend.models._enums` classes.
- `*_id` foreign keys以后要填写的字段。
- `mypy backend/schemas/` must pass with zero errors.
- `pytest backend/tests/test_schemas.py` must pass before any commit.

from __future__ import annotations

from pydantic import BaseModel, ConfigDict, Field


class BaseSchema(BaseModel):
    """Common settings for all schemas."""

    model_config = ConfigDict(str_strip_whitespace=True, extra="forbid")


class BaseCreateSchema(BaseSchema):
    """Base for request (CREATE/UPDATE) schemas."""

class BaseResponseSchema(BaseSchema):
    """Base for response schemas with ORM-to-Pydantic support."""

    model_config = ConfigDict(from_attributes=True, str_strip_whitespace=True, extra="forbid")
