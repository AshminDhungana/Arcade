# Arcade Developer Guide

> Last updated: 2026-07-02 (Phase 1, in progress)
> Target: Python 3.11+, Node.js 20 LTS

## Prerequisites

- **Python 3.11+** (all OSes supported)
- **Node.js 20 LTS** (from [nodejs.org](https://nodejs.org/))
- **Git**
- (Optional) **Make** (for convenience; individual commands are shown below)

## Quick Start

```bash
git clone <repo-url> arcade
cd arcade
make install   # Installs Python deps + frontend deps + agent deps
```

`make install` runs:
- `pip install -r backend/requirements.txt`
- `npm install` in `frontend/` and `agent/`

## Backend Setup

```bash
cd backend
python -m venv venv
source venv/bin/activate      # macOS/Linux
# or: venv\Scripts\activate   # Windows

pip install -r requirements.txt -r requirements-dev.txt
```

### Start the FastAPI dev server

```bash
# Option A: Make target (recommended)
make backend-dev

# Option B: Direct
python -m uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

**Dev endpoints:**
- API: `http://localhost:8000/api`
- Health: `http://localhost:8000/health`
- Swagger: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`

## Frontend Setup

```bash
make frontend-dev       # Or: cd frontend && npm install && npm run dev
```

**Dev endpoints:** `http://localhost:5173` (Vite proxies `/api` -> `localhost:8000`, `/ws` -> `ws://localhost:8000`).

## Agent Setup

```bash
make agent-dev       # Or: cd agent && npm install && npm start
```

## Testing

### Backend

```bash
cd backend
python -m pytest -v

# Single test
pytest tests/test_database.py::test_wal_mode -v

# With coverage
pytest --cov=backend --cov-report=term-missing
```

### Frontend

```bash
cd frontend && npm test -- --run
```

## Linting

### Python

```bash
ruff check --fix backend/
ruff format backend/
mypy --strict backend/
```

Configured in `pyproject.toml`: Ruff rules `E`, `F`, `I`, `UP`, `B`, `S`; Black (88-char line); MyPy `strict = true`.

### Frontend & Agent

```bash
cd frontend && npm run lint
cd agent && npm run lint
```

## Database Migrations

Run all Alembic commands from inside `backend/`:

```bash
cd backend

# Apply all pending migrations
alembic upgrade head

# Generate a new migration after changing a model
alembic revision --autogenerate -m "add <description>"

# Rollback one step
alembic downgrade -1

# Show current migration
alembic current
```

## Seed Script

After `alembic upgrade head`:

```bash
cd backend
python -m scripts.seed_dev
```

**Populates:** 2 zones, 8 seats, 2 staff (Admin & Cashier, both PIN `0000`), 5 menu items, 3 members, 10 feature flags.

**Warning:** `clear_data()` removes all existing data (except `alembic_version`). Only for dev DB.

## Configuration

### `arcade.config.json` (Server-side)

Created by the setup wizard on first run. Never commit this file. On Linux/macOS: `chmod 600 arcade.config.json`.

### `agent.config.json` (Client-side)

Generated by the setup wizard; placed on each client machine. Contains `agent_secret` and `seat_id`. On Linux/macOS: `chmod 600 agent.config.json`.

## Summary of `make` Targets

| Command | Purpose |
|---------|---------|
| `make install` | Install all deps (Python + Node.js) |
| `make backend-dev` | Start FastAPI with hot reload |
| `make frontend-dev` | Start Vite dev server |
| `make agent-dev` | Start Electron in dev mode |
| `make test` | Run all backend + frontend tests |
| `make lint` | Run all linters |
| `make lint-python` | Run ruff + mypy |
| `make build-frontend` | Production build (frontend/dist/) |
| `make build-agent` | Production build (agent/dist/) |
| `make clean` | Clean build artifacts |

## License for Development

During development, you do not need a valid `license.key`:
- `make backend-dev` directly starts the FastAPI server
- The Tkinter Launcher is required only for production deployments
- To test the licensing flow: use `python tools/keygen/generate_keys.py` (see `docs/security/key-management.md`)
