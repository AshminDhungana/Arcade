# License Keygen GUI — Refined Brand Console (Design Spec)

**Date:** 2026-07-19
**Status:** Approved (design)
**Scope:** Visual redesign of the Arcade offline license-generator GUI.
**File changed:** `tools/keygen/license_gui.py` (presentation layer only).

## Goal

Make the internal `license_gui.py` (CustomTkinter) desktop window look like a
professional, on-brand product instead of a bare utility form, while leaving the
cryptographic signing path byte-for-byte identical. Apply UI/UX and visual-taste
principles within the constraints of a Tkinter desktop app.

## Non-goals

- No change to signing logic in `generate_license.py` (`build_and_write_license`,
  `parse_trial_days`, `format_verify_command`, `KeygenError`). License output is
  deterministic and unchanged.
- No framework change (stays CustomTkinter; no web/Electron rewrite).
- No custom widget kit / floating-label components (selected against in favor of
  scope and maintainability).

## Decisions locked during brainstorming

1. **Approach:** Refined Brand Console — elevate the existing single-column card
   with a branded header, real type scale, 8px spacing rhythm, custom success
   toast, inline validation, rasterized logo, and verified-contrast themes.
2. **Framework:** Keep CustomTkinter (offline internal tool; no new runtime deps).
3. **Logo rendering:** Pre-render PNGs from the SVG and commit them. Runtime only
   needs Pillow (already available via CustomTkinter).
4. **Polish dimensions (all four):** branded header & identity, typography &
   spacing, micro-interactions, theme & contrast.

## Architecture

```
generate_license.py  (signing logic — UNTOUCHED, lazy-imports launch_gui)
        │  imports
        ▼
license_gui.py       (REWRITTEN — presentation only)
        │  loads
        ▼
icon/arcade_logo_64.png, arcade_logo_128.png   (committed assets)
        ▲  generated by
icon/render_logo.mjs   (dev-only sharp script; not a runtime dependency)
```

- `license_gui.py` keeps the same public entry point `launch_gui()` so
  `generate_license.main()` continues to lazy-import it.
- `generate_license.py` needs no behavioral change; only a doc comment may be
  tidied.
- Assets are committed, so running the tool requires no Node/Sharp install.

## Components / sections

### Section 1 — Architecture & untouched logic
- Signing helpers imported exactly as today (`KeygenError`, `build_and_write_license`,
  `format_verify_command`, `parse_trial_days`).
- Only `license_gui.py` is rewritten. New committed PNG assets. No new runtime deps.

### Section 2 — Layout & branded identity
- Scrollable single column, max content width ~600px (prevents clipping on small
  windows after the richer content is added).
- **Header band:** 64px logo (left) + two-line lockup ("Arcade" bold, "License
  Generator" muted) + a small "Internal tool" pill and the appearance toggle
  (right). This is the primary identity upgrade.
- **8px spacing rhythm** (8/16/24); consistent 40px field height; section label
  inside the card.
- **Result card hidden until first generate** — cleaner empty state; window opens
  focused on the form.
- Layout (ASCII):
  ```
  ┌──────────────────────────────────────────────┐
  │  [logo]  Arcade                                │  ← header band
  │          License Generator                      │
  │          Issue a signed license key ·  Internal │
  ├──────────────────────────────────────────────┤
  │  ┌── Card: License details ──────────────┐    │
  │  │  Hardware ID *          [___________]  │    │
  │  │  helper text                            │    │
  │  │  Cafe Name *            [___________]  │    │
  │  │  License Type   [ Perpetual | Trial ]  │    │
  │  │  Trial Days             [___________]  │    │
  │  │  Output File   [__________] [Browse]   │    │
  │  └────────────────────────────────────────┘    │
  │        [   Generate License   ]                 │  ← primary, full width
  │  ✓ toast / status line                          │
  │  ┌── Result (only after generate) ───────┐     │
  │  │  path • monospace blob • verify cmd     │    │
  │  │  [Copy License] [Copy Verify]           │    │
  │  └─────────────────────────────────────────┘   │
  └──────────────────────────────────────────────┘
  ```

### Section 3 — Typography, spacing & theme/contrast
- **Type scale** (explicit px in one constants block):
  - Display (app title) 24 bold · Card heading 15 semibold · Field label 13
    semibold · Helper/body 12 · Entry text 13 · Button 14 semibold · Mono blob
    11 Courier New.
- **Spacing:** single `PAD = 8` base, used 2×/3×; card padding 24; field-group
  gap 16.
- **Theme & contrast (WCAG AA verified):**
  - Light: bg `#F8FAFC`, card `#FFFFFF` + border `#E2E8F0`; text `#0F172A`
    (≈13:1), body `#334155` (≈10:1), muted `#64748B` (≈5.3:1); primary `#2563EB`
    + white (≈4.6:1).
  - Dark: bg `#0F172A`, card `#1E293B` + border `#334155`; text `#F8FAFC`
    (≈13:1), body `#CBD5E1` (≈10:1), muted `#94A3B8` (≈5.9:1); primary `#3B82F6`
    + near-black text (≈5.9:1).
  - System/Dark/Light toggle retained. Brand accent gradient (`#0334F0 →
    #06A3FC`, sampled from the logo) used **only** as a thin 3px header underline
    and the success checkmark — reads as brand without visual noise.

### Section 4 — Micro-interactions & states
- **Success toast:** custom in-window frame slides in from top-right with a fade
  (via `after()` stepping alpha/offset; no async), shows a green check badge +
  "License generated", auto-dismisses after 3s. Persistent status line still used
  for errors.
- **Inline validation:** on Generate, empty required fields (Hardware ID, Cafe
  Name) get a red border ring + one-line error message under the field; focus
  jumps to first invalid field; cleared on edit. Non-numeric trial days use the
  same treatment.
- **Button feedback:** primary button shows disabled "Generating…" state during
  the sign call and restores on completion.
- **Hover/press:** native CustomTkinter hover; primary hover `#1D4ED8`/`#2563EB`
  (light), `#2563EB`/`#3B82F6` (dark). Copy → transient "Copied ✓" label.
- **Copy buttons:** keep License-string / Verify-command; on click set clipboard
  then flip label to "Copied ✓" for ~1.2s.

### Section 5 — Error handling, assets & testing
- **Error handling:** `KeygenError` → red status line + `messagebox.showerror`
  (unchanged behavior, restyled surroundings). Missing logo PNG → app still
  launches with a text wordmark fallback (no crash).
- **Asset generation:** committed `tools/keygen/icon/render_logo.mjs` (validated
  `sharp` script) documents how the PNGs are produced from the SVG. `sharp` is
  dev-only and **not** added to runtime requirements. PNGs are committed so no
  Node install is needed to run the tool.
- **`requirements.txt`:** ensure `Pillow` is explicitly listed (CustomTkinter
  depends on it; make the dependency explicit rather than implicit).
- **Testing:**
  - Existing `generate_license.py` logic tests pass unchanged (signer untouched).
  - New pure helpers extracted and unit-tested without a display:
    - `validate_inputs(hardware_id, cafe_name, license_type, trial_days) -> dict[field, error]`
    - `resolve_logo_path() -> Path | None` (fallback when PNG missing).
  - Manual smoke checklist (in spec, run during implementation): launch; light/
    dark toggle; empty-field validation; perpetual generate; trial generate; both
    copy buttons; missing-PNG fallback.

## Data flow (unchanged semantics)

1. User fills form → clicks Generate.
2. `validate_inputs(...)` returns error map; if non-empty, apply inline errors and
   abort.
3. Else call `build_and_write_license(...)` → base64 blob (and
   `format_verify_command(...)`).
4. On `KeygenError` → error status + messagebox. On success → populate result
   card, show success toast, enable copy buttons.

## Acceptance criteria

- App launches offline with no new runtime dependencies (Pillow already required).
- Logo renders in the header in both light and dark modes.
- Success path shows the toast; error/empty paths show inline validation + status
  line; no crash if logo PNG is missing.
- Generated `license.key` content is byte-identical to the pre-redesign output for
  the same inputs.
- Existing signing-logic tests still pass.
